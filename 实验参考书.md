## 一、实验背景与原理

### 1.1 PySpark简介

PySpark是Apache Spark的Python API，它允许使用Python进行大规模数据处理和分析。PySpark基于Py4J库实现，该库使Python能够动态地与JVM对象交互。

**核心组件**：
- **SparkContext**：Spark应用的入口，负责与集群连接
- **SparkSession**：Spark 2.0+推荐的统一入口，封装了SparkContext、SQLContext和HiveContext
- **RDD（弹性分布式数据集）**：Spark的基础数据结构，支持并行操作
- **DataFrame/Dataset**：结构化数据API，提供更高效的执行优化

### 1.2 WordCount原理

WordCount是大数据处理的"Hello World"程序，其核心流程为：
1. **读取文本** → 从HDFS或本地文件系统加载数据
2. **分词处理** → 将每行文本拆分为单词
3. **映射转换** → 将单词映射为`(word, 1)`键值对
4. **归约统计** → 按Key聚合，统计每个单词出现次数
5. **输出结果** → 保存或展示统计结果

## 三、实验核心代码详解

### 3.1 完整WordCount程序（exp2.py）

```python
from pyspark.sql import SparkSession
import sys

def word_count(input_path, output_path):
    """
    PySpark WordCount程序
    功能：统计文本文件中各单词出现频率
    """
    
    # Step 1: 初始化SparkSession
    # SparkSession是Spark 2.0+的统一入口，封装了SparkContext
    spark = SparkSession.builder \
        .appName("PythonWordCount") \  # 设置应用名称，显示在Web UI中
        .master("local[*]") \          # 本地模式，[*]使用所有CPU核心
        .getOrCreate()                 # 获取或创建新的SparkSession
    
    # 获取SparkContext（底层API入口）
    sc = spark.sparkContext
    
    # Step 2: 读取文本文件
    # textFile()将文件加载为RDD，每行作为一个元素
    lines = sc.textFile(input_path)
    
    # Step 3: 数据转换与处理
    
    # 3.1 分词：flatMap将每行拆分为单词，扁平化为单个单词流
    # lambda函数：对每行line执行split操作
    words = lines.flatMap(lambda line: line.split(" "))
    
    # 3.2 过滤空字符串（可选但推荐）
    words = words.filter(lambda word: len(word) > 0)
    
    # 3.3 转换为键值对：map将每个word转为(word, 1)
    word_pairs = words.map(lambda word: (word, 1))
    
    # 3.4 归约统计：reduceByKey按Key聚合，将相同word的count相加
    # lambda x, y: x + y 定义了聚合逻辑（累加）
    word_counts = word_pairs.reduceByKey(lambda x, y: x + y)
    
    # Step 4: 结果处理
    
    # 4.1 按词频降序排序（可选）
    # sortBy是转换操作（Transformation），惰性执行
    sorted_counts = word_counts.sortBy(lambda x: x[1], ascending=False)
    
    # 4.2 触发执行并收集结果（Action操作）
    # collect()将RDD数据拉取到Driver端内存（仅适用于小数据量）
    results = sorted_counts.collect()
    
    # Step 5: 输出结果
    print("\n" + "="*40)
    print("Word Count Results (Top 20):")
    print("="*40)
    for word, count in results[:20]:
        print(f"{word:<15} : {count:>5}")
    
    # 保存到文件（会产生part-00000等文件）
    word_counts.saveAsTextFile(output_path)
    
    # Step 6: 关闭SparkSession
    spark.stop()
    print(f"\nResults saved to: {output_path}")

if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("Usage: python exp2.py <input_path> <output_path>")
        sys.exit(1)
    
    input_file = sys.argv[1]
    output_dir = sys.argv[2]
    word_count(input_file, output_dir)
```

### 3.2 关键API解析

| API | 类型 | 作用 | 理论课对应概念 |
|-----|------|------|----------------|
| `SparkSession.builder` | 入口 | 构建Spark会话 | Driver Program |
| `textFile()` | Transformation | 读取文件为RDD | Data Loading |
| `flatMap()` | Transformation | 扁平化映射 | FlatMap Operation |
| `map()` | Transformation | 一对一转换 | Map Operation |
| `filter()` | Transformation | 条件过滤 | Filter Operation |
| `reduceByKey()` | Transformation | 按Key归约 | Shuffle & Reduce |
| `sortBy()` | Transformation | 排序 | Sorting |
| `collect()` | Action | 收集结果到Driver | Trigger Execution |
| `saveAsTextFile()` | Action | 保存到文件 | Output Writing |

**Transformation vs Action**：
- **Transformation**（转换）：惰性执行，仅记录操作逻辑，不立即计算，返回新的RDD
- **Action**（动作）：触发实际计算，将结果返回Driver或写入外部存储

### 5.3 WordCount执行流程分析

**在Web UI中观察到的典型流程**：

1. **Job 0**（由`collect()`触发）：
   - Stage 0：`textFile` → `flatMap` → `map` → `filter`（无Shuffle）
   - Stage 1：`reduceByKey`（宽依赖，触发Shuffle）
   - Stage 2：`sortBy`（可能触发额外Shuffle）

2. **Job 1**（由`saveAsTextFile()`触发）：
   - Stage 3：文件写入操作

**关键观察点**：
- **DAG Visualization**：查看有向无环图，理解RDD依赖关系
- **Summary Metrics**：检查Min/Median/Max差异，识别数据倾斜（如某Task耗时远超其他）

## 六、常见问题与解决方案

### 6.1 Python版本问题

**症状**：`Exception: Python in worker has different version 3.8 than that in driver 3.9`

**解决**：确保所有节点PYSPARK_PYTHON指向相同Python版本，或在`spark-submit`中指定：
```bash
--conf "spark.pyspark.python=/path/to/python3.9"
```

### 6.2 权限问题

**症状**：`Permission denied: /tmp/spark-events`

**解决**：
```bash
sudo chmod 777 /tmp/spark-events
# 或更改日志目录到用户目录
mkdir -p ~/spark-logs
export SPARK_LOG_DIR=~/spark-logs
```

### 6.3 端口冲突

**症状**：`Failed to bind to: /0.0.0.0:4040`

**解决**：Spark会自动尝试下一个端口，或手动指定：
```bash
--conf "spark.ui.port=4041"
```

### 6.4 内存不足

**症状**：`java.lang.OutOfMemoryError: Java heap space`

**解决**：调整Driver/Executor内存：
```bash
spark-submit \
    --driver-memory 2g \
    --executor-memory 2g \
    exp2.py
```
